# 發佈說明 - Updater v1.1.1 (最新)

## 摘要
本次更新專注於提升使用者體驗，加入了版本顯示功能，並針對 Windows 高解析度螢幕優化了 GUI 視窗的顯示效果與啟動位置。

## 新增功能與改進

### 1. 啟動版本顯示
*   **CLI 模式**：現在執行時會在最上方優先顯示版本號。
*   **GUI 模式**：啟動後的日誌主控台 (Log Console) 會第一時間印出當前版本。
*   **智慧識別**：開發環境會顯示 `vdevelop version`，打包後的 EXE 則會顯示正式版本號（如 `v1.1.1`）。

### 2. GUI 視窗優化 (High-DPI 支援)
*   **DPI 自適應**：支援 Windows 系統縮放（例如 125%、150%），解決視窗模糊與比例失調問題。
*   **動態高度**：視窗高度現在會自動調整為螢幕邏輯高度的 **70%**。
*   **智慧定位**：視窗啟動時會自動處於螢幕 **水平置中**，且垂直位置位於頂部 **3%** 處，視覺感官更平衡。

### 3. 自動化建置流程
*   新增版本同步腳本，確保 `pyproject.toml` 中的版本號會自動硬寫進 EXE 檔案中，無需手動修改程式碼。

# 發佈說明 - Updater v1.1.0

## 摘要
引入外部設定檔支援 (`updater.toml`)，無需重新編譯即可修改套件來源路徑與管理清單。

## 主要變更
*   **設定檔支援**：支援讀取 `updater.exe` 同目錄下的 `updater.toml`。
*   **動態配置**：可透過編輯設定檔直接修改 `source` 路徑或增減 `packages` 套件。
*   **相容性**：若設定檔不存在，將自動退回使用內建的預設值（PNT52 路徑）。

# 發佈說明 - Updater v1.0.0

## 摘要
Updater 微服務正式從 `test_matrix` 主專案中獨立出來，成為具備獨立虛擬環境、版本控制與建置流程的子專案。

## 主要功能
*   **微服務架構**：獨立的運行環境與依賴管理。
*   **效能提升**：採用批次 `pip` 查詢技術，大幅縮短掃描與比對時間。
*   **Nuitka 打包**：編譯為獨立 EXE，目標電腦無需安裝 Python 環境。
*   **PEP 440 標準**：嚴格遵循 Python 官方版本比對規範。
