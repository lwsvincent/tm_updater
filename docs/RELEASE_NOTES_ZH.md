# 發佈說明 - Updater v1.2.0 (最新)

## 摘要
本次更新重點推出功能完整的 GUI 啟動器，支援圖形化套件管理與自動更新流程，同時完成了代碼架構重構，提升可維護性與測試覆蓋率。

## 新增功能與改進

### 1. GUI 啟動器 (PyWebView)
*   **Vue 前端框架**：現代化的圖形介面，提供套件管理與實時監控。
*   **即時狀態顯示**：套件清單、版本比對與更新進度一目瞭然。
*   **日誌流傳輸**：實時顯示後端指令輸出，支援詳細操作追蹤。
*   **PyWebView 橋接**：無縫整合 GUI 與後端邏輯。

### 2. 自動更新流程
*   **配置驅動啟動**：支援外部配置檔控制初始化行為。
*   **自動掃描檢測**：後台自動檢查可用更新。
*   **預覽與執行**：支援 dry-run 模式預覽更新，確認後再執行。
*   **更新後啟動**：自動偵測執行環境模式（CLI/GUI），適當啟動應用。

### 3. 建置工具改進
*   **Nuitka 編譯**：生成完全獨立的 `.exe` 可執行檔。
*   **分離構建腳本**：`build_gui.bat` 與 `build_updater.bat` 分別編譯 GUI 和 CLI。
*   **完整測試涵蓋**：CLI、配置、核心模組、啟動器、GUI 的綜合測試套件。

### 4. 代碼架構重構
*   **核心邏輯提取**：獨立的掃描、比對、安裝模組。
*   **配置管理**：類型化配置模組，支援 TOML 檔案加載。
*   **啟動器模組**：專職處理更新後的應用啟動邏輯。
*   **薄層 CLI 包裝**：Main 模組現為簡潔的命令列介面。

## 升級說明

GUI 啟動器為選擇性功能，既有的 CLI 工作流程完全相容，無需修改。若要使用 GUI，請確保環境中已安裝 `pywebview>=5.0` 版本。

# 發佈說明 - Updater v1.1.1 (最新)

## 摘要
本次更新專注於提升使用者體驗，加入了版本顯示功能，並針對 Windows 高解析度螢幕優化了 GUI 視窗的顯示效果與啟動位置。

## 新增功能與改進

### 1. 啟動版本顯示
*   **CLI 模式**：現在執行時會在最上方優先顯示版本號。
*   **GUI 模式**：啟動後的日誌主控台 (Log Console) 會第一時間印出當前版本。
*   **智慧識別**：開發環境會顯示 `vdevelop version`，打包後的 EXE 則會顯示正式版本號（如 `v1.1.1`）。

### 2. GUI 視窗優化 (High-DPI 支援)
*   **DPI 自適應**：支援 Windows 系統縮放（例如 125%、150%），解決視窗模糊與比例失調問題。
*   **動態高度**：視窗高度現在會自動調整為螢幕邏輯高度的 **70%**。
*   **智慧定位**：視窗啟動時會自動處於螢幕 **水平置中**，且垂直位置位於頂部 **3%** 處，視覺感官更平衡。

### 3. 自動化建置流程
*   新增版本同步腳本，確保 `pyproject.toml` 中的版本號會自動硬寫進 EXE 檔案中，無需手動修改程式碼。

# 發佈說明 - Updater v1.1.0

## 摘要
引入外部設定檔支援 (`updater.toml`)，無需重新編譯即可修改套件來源路徑與管理清單。

## 主要變更
*   **設定檔支援**：支援讀取 `updater.exe` 同目錄下的 `updater.toml`。
*   **動態配置**：可透過編輯設定檔直接修改 `source` 路徑或增減 `packages` 套件。
*   **相容性**：若設定檔不存在，將自動退回使用內建的預設值（PNT52 路徑）。
